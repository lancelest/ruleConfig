# 风控工作台 - 基准偏离监控模块需求文档

## 项目概述

- **项目名称**：风控工作台 - 基准偏离监控
- **平台**：Web端 (1920×1080)
- **所属公司**：天弘基金
- **页面数量**：5个核心页面 + 1个组件页

## 核心数据模型

### 关系：组合 × 指标 × 组合类型 = 规则配置

```
组合(基金产品) ←→ 组合基准 ←→ 基准 ←→ 指标规则 ←→ 指标
                      ↓
               每日计算偏离值
                      ↓
               偏离值 vs 阈值 → 触发预警(L1/L2/L3/L4)
```

### 4级预警阈值

| 等级 | 名称 | 颜色 | 含义 |
|------|------|------|------|
| L1   | 正常 | 绿色 | 阈值内 |
| L2   | 警告 | 黄色 | 轻度超阈值 |
| L3   | 错误 | 橙色 | 中度超阈值 |
| L4   | 严重 | 红色 | 重度超阈值 |

## 规则引擎设计（v11 新增）

### 整体业务流程

```
┌─────────┐    ┌─────────┐    ┌──────────────────┐    ┌──────────────┐
│  数仓    │───→│ Java后端 │───→│   触发事件表      │───→│   前端展示   │
│ (上游)   │    │  (规则引擎)│    │                  │    │  (下游)     │
└─────────┘    └─────────┘    └──────────────────┘    └──────────────┘
     │              │                │
  ① 跑批计算      ② 对照规则引擎    ③ 写入触发事件
  指标实际值       判定预警等级      (L1/L2/L3/L4)
```

**三个岗位职责分界：**
- **数仓**：根据规则配置，跑批计算每个基金产品的各项指标实际值
- **Java 后端**：拿指标计算结果，对照规则引擎判定预警等级，写入触发事件表
- **前端**：展示触发事件、处理流程

### 规则编码生成规则

```
规则编码 (ruleCode) = 产品维度 + 指标维度

例如：
  RULE_EXCESS_RET_AE   →  主动权益 + 超额收益
  RULE_STOCK_POS_AE    →  主动权益 + 股票仓位偏离
  RULE_STOCK_POS_II    →  指数增强 + 股票仓位偏离

一条规则下面有：
  - 具体的指标定义（metricCode、metricName）
  - 触发条件（operator + threshold）
  - AND 条件组合（多个条件联合判定）
  - 4 级预警阈值（level1 ~ level4）
  - 匹配的基金产品列表（N 个基金产品共享同一条规则）
```

### 细分类别设计

**场景**：固收+的股票仓位偏离按基准比例分档，不同档位阈值不同

**方案**：独立表 `rule_sub_category`，支持动态扩展

```
一条规则（ruleCode）下可能有多条细分类别配置
如果该规则没有细分类别 → rule_sub_category 表无记录，直接用 rule_config 表的阈值
如果该规则有细分类别 → 查 rule_sub_category 表获取各档位阈值
```

### 豁免组合设计

**场景**：某些组合类型对特定规则/层级不触发预警

**方案**：独立表 `rule_exemption`，豁免到组合类型级别，带开关控制

```
一个规则可能豁免多个组合类型：
  RULE_STOCK_POS_AE 的 L1 豁免：固收+、固收
  RULE_STOCK_POS_AE 的 L2 豁免：固收
```

### 双向查询兼容

**同一套规则，两种查询视角：**

```
组合类型视角（默认）
  请求参数: portTypeCode=AE
  查询逻辑: SELECT * FROM rule_config WHERE port_type_code = 'AE'

指标视角
  请求参数: metricCode=excess_ret_1y
  查询逻辑: SELECT * FROM rule_config WHERE metric_code = 'excess_ret_1y'
```

### 规则引擎核心逻辑

```
规则引擎评估流程：
1. 根据 portTypeCode + metricCode 查找匹配的规则
2. 检查该规则是否被豁免
3. 如有细分类别，根据组合基准值匹配对应的阈值配置
4. 比较 metricValue 与阈值，判定触发等级
5. 返回评估结果 → 写入触发事件表
```

### 阈值类型

| 类型 | 示例 | 比较逻辑 |
|------|------|---------|
| 负向（超额收益） | -5%, -7%, -10% | `value <= threshold` 触发，越负等级越高 |
| 偏离（绝对值） | 15%, 20%, 25% | `|value| >= threshold` 触发，越大等级越高 |
