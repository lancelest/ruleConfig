# 预警规则配置 - 数据库设计

## 设计原则

- **最小字段**：只建确定需要的字段，后续扩展加列是秒级操作
- **JSON存规则条件**：多指标AND组合放JSON字段，灵活可扩展
- **operator存英文标识**：如 `GT`，避免JSON转义问题，语义清晰
- **组合维度与指标维度共用一张表**：两种维度只是查询视角不同，底层结构一致

---

## 核心表结构

### 1. 指标元数据表 (indicator_metadata)

```sql
CREATE TABLE indicator_metadata (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    indicator_code  VARCHAR(64) NOT NULL UNIQUE COMMENT '指标编码，如：excess_return_1y',
    indicator_name  VARCHAR(128) NOT NULL COMMENT '指标名称，如：超额收益_近一年',
    category_code   VARCHAR(64) NOT NULL COMMENT '指标分类编码，如：performance',
    category_name   VARCHAR(64) COMMENT '指标分类名称，如：业绩表现',
    calc_formula    TEXT COMMENT '指标计算逻辑说明',
    create_time     DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time     DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_category (category_code)
) COMMENT='指标元数据表';
```

**示例数据**：
| indicator_code | indicator_name | category_code | category_name |
|---------------|----------------|---------------|---------------|
| excess_return_1y | 超额收益_近一年 | performance | 业绩表现 |
| max_drawdown_1y | 最大回撤_近一年 | performance | 业绩表现 |
| stock_position_deviation | 股票仓位偏离 | equity_allocation | 权益资产配置 |
| benchmark_risk_ratio | 基准风险比例 | equity_allocation | 权益资产配置 |

### 2. 规则配置表 (warning_rule_config)

组合维度和指标维度共用一张表，通过查询条件区分视角

```sql
CREATE TABLE warning_rule_config (
    id                  BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    rule_code           VARCHAR(64) NOT NULL COMMENT '规则编码，业务唯一标识',
    rule_name           VARCHAR(128) NOT NULL COMMENT '规则名称',
    
    -- 组合类型（组合维度时作为主筛选条件，指标维度时作为附加属性）
    portfolio_type_code VARCHAR(64) COMMENT '组合类型编码，如：active_equity',
    portfolio_type_name VARCHAR(64) COMMENT '组合类型名称，如：主动权益',
    
    -- 指标信息（指标维度时作为主筛选条件，组合维度时作为附加属性）
    indicator_code      VARCHAR(64) COMMENT '指标编码',
    indicator_name      VARCHAR(128) COMMENT '指标名称',
    
    -- 细分类别（如：基准股票比例10%~20%）
    sub_category        VARCHAR(128) COMMENT '细分类别',
    
    -- 规则条件（JSON格式，支持多指标AND组合）
    rule_conditions     JSON NOT NULL COMMENT '规则条件定义，见下方JSON结构',
    
    -- 4级预警阈值
    threshold_l1        DECIMAL(18,6) COMMENT '层级1阈值（正常边界）',
    threshold_l2        DECIMAL(18,6) COMMENT '层级2阈值（警告）',
    threshold_l3        DECIMAL(18,6) COMMENT '层级3阈值（错误）',
    threshold_l4        DECIMAL(18,6) COMMENT '层级4阈值（严重）',
    
    -- 版本管理
    version             INT NOT NULL DEFAULT 1 COMMENT '版本号',
    status              VARCHAR(32) NOT NULL DEFAULT 'DRAFT' COMMENT '状态：DRAFT-草稿, ACTIVE-生效, INACTIVE-停用, ARCHIVED-归档',
    is_current_version  TINYINT DEFAULT 0 COMMENT '是否当前生效版本：0-否，1-是',
    deleted             TINYINT DEFAULT 0 COMMENT '是否删除：0-未删除，1-已删除（伪删除）',
    
    -- 生效时间
    effective_time      DATETIME COMMENT '生效时间',
    expire_time         DATETIME COMMENT '失效时间',
    
    -- 创建信息
    create_by           VARCHAR(64) COMMENT '创建人',
    create_time         DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_by           VARCHAR(64) COMMENT '更新人',
    update_time         DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    -- 索引
    UNIQUE KEY uk_rule_version (rule_code, version),
    INDEX idx_portfolio (portfolio_type_code, status, is_current_version),
    INDEX idx_indicator (indicator_code, status, is_current_version),
    INDEX idx_status (status, is_current_version),
    INDEX idx_effective (effective_time, expire_time)
    
) COMMENT='预警规则配置表';
```

**两种维度查询方式**：
```sql
-- 组合维度：按组合类型查该类型下所有指标规则
SELECT * FROM warning_rule_config 
WHERE portfolio_type_code = 'active_equity' 
  AND is_current_version = 1 AND status = 'ACTIVE';

-- 指标维度：按指标查该指标在所有组合类型下的规则
SELECT * FROM warning_rule_config 
WHERE indicator_code = 'excess_return_1y' 
  AND is_current_version = 1 AND status = 'ACTIVE';
```

### 3. 规则条件 JSON 结构设计

**operator 取值**：
| 英文标识 | 含义 | 说明 |
|---------|------|------|
| GT | 大于 | Greater Than |
| GTE | 大于等于 | Greater Than or Equal |
| LT | 小于 | Less Than |
| LTE | 小于等于 | Less Than or Equal |
| ABS_GT | 绝对值大于 | Absolute Greater Than |
| ABS_LT | 绝对值小于 | Absolute Less Than |

**单指标规则示例**：
```json
{
  "logic": "SINGLE",
  "conditions": [
    {
      "indicatorCode": "excess_return_1y",
      "indicatorName": "超额收益_近一年",
      "operator": "GT",
      "threshold": 0.05
    }
  ]
}
```

**多指标AND组合规则示例**（场景：超额收益率大于0 且 基准风险比例小于10% 且 整体仓位偏离大于5%）：
```json
{
  "logic": "AND",
  "conditions": [
    {
      "indicatorCode": "excess_return_1y",
      "indicatorName": "超额收益_近一年",
      "operator": "GT",
      "threshold": 0
    },
    {
      "indicatorCode": "benchmark_risk_ratio",
      "indicatorName": "基准风险比例",
      "operator": "LT",
      "threshold": 0.10
    },
    {
      "indicatorCode": "stock_position_deviation",
      "indicatorName": "整体仓位偏离",
      "operator": "GT",
      "threshold": 0.05
    }
  ]
}
```

### 4. 触发事件表 (warning_trigger_event)

```sql
CREATE TABLE warning_trigger_event (
    id                  BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    event_no            VARCHAR(64) NOT NULL UNIQUE COMMENT '事件编号，业务唯一',
    
    -- 关联规则
    rule_id             BIGINT NOT NULL COMMENT '规则配置ID',
    rule_code           VARCHAR(64) NOT NULL COMMENT '规则编码',
    rule_version        INT NOT NULL COMMENT '触发时规则版本',
    
    -- 触发对象
    portfolio_code      VARCHAR(64) NOT NULL COMMENT '组合代码（基金代码）',
    portfolio_name      VARCHAR(128) COMMENT '组合名称',
    portfolio_type_code VARCHAR(64) COMMENT '组合类型',
    
    -- 触发结果
    trigger_level       VARCHAR(16) NOT NULL COMMENT '触发等级：L1, L2, L3, L4',
    trigger_values      JSON COMMENT '各指标实际值（JSON）',
    
    -- 计算日期
    calc_date           DATE NOT NULL COMMENT '计算日期（T日）',
    data_snapshot       JSON COMMENT '触发时的数据快照（用于追溯）',
    
    -- 处置信息（处置流）
    dispose_status      VARCHAR(32) DEFAULT 'PENDING' COMMENT '处置状态：PENDING-待处置, DISPOSED-已处置',
    dispose_time        DATETIME COMMENT '处置时间',
    dispose_by          VARCHAR(64) COMMENT '处置人',
    dispose_opinion     TEXT COMMENT '处置意见',
    
    -- 回复信息（回复流）
    reply_status        VARCHAR(32) DEFAULT 'PENDING' COMMENT '回复状态：PENDING-待回复, REPLIED-已回复',
    reply_time          DATETIME COMMENT '回复时间',
    reply_by            VARCHAR(64) COMMENT '回复人',
    reply_content       TEXT COMMENT '回复内容',
    
    create_time         DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time         DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_rule (rule_code, rule_version),
    INDEX idx_portfolio (portfolio_code, calc_date),
    INDEX idx_status (dispose_status, reply_status),
    INDEX idx_calc_date (calc_date),
    INDEX idx_trigger_level (trigger_level)
    
) COMMENT='预警触发事件表';
```

**trigger_values JSON示例**（多指标时）：

    "value": 0.08,
    "threshold": 0.10,
    "operator": "LT",
    "hit": true
  },
  "stock_position_deviation": {
    "value": 0.06,
    "threshold": 0.05,

  }

### 5. 事件通知表 (warning_event_notification)

```sql
CREATE TABLE warning_event_notification (
    id                  BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    
    -- 关联事件
    event_id            BIGINT NOT NULL COMMENT '触发事件ID',
    
    -- 通知配置
    notify_type         VARCHAR(32) NOT NULL COMMENT '通知类型：EMAIL-邮件, TODO-待办',
    notify_title        VARCHAR(256) COMMENT '通知标题',
    
    -- 接收人
    receiver_id         VARCHAR(64) NOT NULL COMMENT '接收人ID',
    
    -- 发送状态
    send_status         VARCHAR(32) DEFAULT 'PENDING' COMMENT '发送状态：PENDING-待发送, SUCCESS-成功, FAILED-失败',
    retry_count         INT DEFAULT 0 COMMENT '重试次数',
    error_msg           TEXT COMMENT '失败原因',
    
    create_time         DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time         DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_event (event_id),
    INDEX idx_notify (notify_type, send_status),
    INDEX idx_receiver (receiver_id)
    
) COMMENT='事件通知记录表';
```

## 版本管理

### 规则保存流程

```
1. 用户编辑规则（基于某个版本）
2. 前端提交：ruleCode + 新规则内容
3. 后端处理：
   a. 查询该 ruleCode 的最大 version
   b. 新 version = max + 1
   c. 插入新记录，status=DRAFT
   d. 用户点击"生效"：
      - 旧版本 is_current_version=0, status=ARCHIVED
      - 新版本 is_current_version=1, status=ACTIVE
```

### 查询当前生效规则

```sql
SELECT * FROM warning_rule_config 
WHERE is_current_version = 1 
  AND status = 'ACTIVE'
  AND (effective_time IS NULL OR effective_time <= NOW())
  AND (expire_time IS NULL OR expire_time >= NOW());
```
